# Annie's Magic Numbers

Profit & margin analysis on Annie's 2016 wholesale liquor data.  Answers
the three questions she asked:

  1. Top 10 products by profit ($) and by margin (%).
  2. Top 10 brands by profit ($) and by margin (%).
  3. Which brands / products to drop because they lose money.

The deliverable is a single Markdown report (`reports/REPORT.md`) plus
the underlying CSVs in `reports/`.  Headline (from a clean run on the
PwC case-study CSVs):

| Revenue | COGS | Excise tax | Net profit | Net margin |
|--------:|-----:|-----------:|-----------:|-----------:|
| $452.1M | $313.4M | $19.0M | $119.7M | 26.5% |

## Repo layout

```
├── analysis/
│   ├── pipeline.sql      ← the analytics: ingest, transform, report, sanity checks
│   └── run.py            ← thin orchestrator: runs the SQL, renders Markdown
├── bin/duckdb.exe        ← DuckDB CLI v1.1.3 (vendored — see "Why DuckDB" below)
├── src/                  ← input CSVs (six files from the PwC case study)
│   ├── 2017PurchasePricesDec.csv
│   ├── BegInvFINAL12312016.csv
│   ├── EndInvFINAL12312016.csv
│   ├── InvoicePurchases12312016.csv
│   ├── PurchasesFINAL12312016.csv
│   └── SalesFINAL12312016.csv
├── reports/              ← generated by `python analysis/run.py`
│   ├── REPORT.md
│   ├── top10_products_by_profit.csv
│   ├── top10_products_by_margin.csv
│   ├── top10_brands_by_profit.csv
│   ├── top10_brands_by_margin.csv
│   ├── drop_candidates_products.csv
│   └── drop_candidates_brands.csv
├── annie.duckdb          ← persisted DB (recreated on every run)
├── .gitignore
└── README.md
```

## Run it

1. Drop the six PwC case-study CSVs into `src/`.
2. Confirm `bin/duckdb.exe` is present (it is — vendored at v1.1.3).
3. Run:

   ```
   python analysis/run.py
   ```
